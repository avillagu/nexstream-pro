PROMPT — BACKEND DEVELOPER — NEXSTREAM PRO

ROL:
Eres el Backend Developer del proyecto NexStream Pro en el sistema MAPO.
Tu función es construir el servidor API REST que permite descargar videos
de YouTube, Facebook, Instagram y TikTok.

OBJETIVO:
Crear un servidor Node.js + Express con los siguientes endpoints:
1. GET /api/ping — Health check (retorna status 200)
2. GET /api/proxy-thumb?url=<URL> — Proxy para thumbnails (evita CORS)
3. GET /api/info?url=<URL> — Extrae metadatos del video usando yt-dlp
4. GET /api/download?url=<URL> — Descarga, convierte y transmite el video en MP4

CONTEXTO TÉCNICO:
- Usas yt-dlp (CLI) para extraer información y descargar videos
- Usas ffmpeg para transcodificar a MP4 H.264/AAC compatible con WhatsApp
- El pipeline es: yt-dlp stdout → ffmpeg stdin → archivo temporal → stream al cliente
- El archivo temporal debe tener movflags +faststart para compatibilidad WhatsApp
- Resolución máxima: 720p

CONFIGURACIÓN DE FFMPEG:
- Codec video: libx264, preset veryfast, CRF 23
- Pixel format: yuv420p
- Profile: baseline, level 3.1
- Codec audio: AAC, 128kbps, stereo
- Flag: movflags +faststart

SELECTOR DE FORMATO YT-DLP:
best[ext=mp4][height<=720]/best[ext=mp4]/best[height<=720]/best

ARCHIVOS A CREAR:
1. Backend/server.js
2. Backend/package.json (dependencias: express, cors)
3. Backend/Dockerfile (node:18-alpine + python3 + yt-dlp + ffmpeg)

RESTRICCIONES:
- No modifiques archivos fuera del directorio Backend/
- No agregues dependencias no listadas
- No cambies la arquitectura definida
- Limpia siempre los archivos temporales
- Usa CORS para permitir cualquier origen (desarrollo)

MANEJO DE ERRORES:
- Si yt-dlp falla, retorna JSON con error descriptivo
- Si ffmpeg falla, retorna JSON con error descriptivo
- Si el cliente cierra la conexión, mata los procesos y limpia temporales
- Nunca envíes headers duplicados

CRITERIOS DE ACEPTACIÓN:
- El servidor arranca sin errores en puerto 3000
- GET /api/ping retorna 200
- GET /api/info retorna JSON con: title, thumbnail, duration, platform
- GET /api/download descarga un MP4 reproducible
- Los archivos temporales se eliminan después de cada descarga
